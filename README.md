# OSA 光谱仪控制程序

基于 PyQt5 的横河 (Yokogawa) AQ6370 系列光谱仪远程控制工具，通过 TCP Socket / SCPI 协议与设备通信，提供图形化操作界面。

经实测验证支持的设备型号：**AQ6370C**（也兼容 AQ6370B 等同系列设备）。

| 项目 | 要求 |


## 功能详解

### 1. 连接设备

| 字段 | 说明 |
|------|------|
| **IP** | 光谱仪的 IP 地址（默认 `192.168.100.176`） |
| **Port** | SCPI 通信端口（默认 `10001`） |
| **连接时重置** | 勾选后连接时会执行 `*RST`（复位）和 `*CLS`（清除状态），会删除当前扫描数据和所有设置 |

操作步骤：
1. 确认电脑与光谱仪在同一局域网，可以 `ping` 通设备 IP
2. 输入正确的 IP 和 Port
3. 点击 **连接**，状态栏显示设备型号即连接成功
4. 连接失败会自动重试一次

点击 **断开** 可主动断开连接。

### 2. 参数设置

配置完成后点击 **应用设置**，参数会一次性下发到设备。所有参数的含义：

| 参数 | 说明 | 有效范围 |
|------|------|---------|
| **扫描模式** | 单次：扫描一次后停止；连续：持续扫描；自动：自动判断 | 单次 / 连续 / 自动 |
| **起始波长** | 扫描起始波长 | 600 ~ 1700 nm |
| **终止波长** | 扫描终止波长 | 600 ~ 1700 nm，且 > 起始波长 |
| **分辨率** | 波长分辨率 | 0.02, 0.05, 0.1, 0.2, 0.5, 1.0, 2.0 nm |
| **纵轴每格刻度** | Y 轴每格的 dB 值 | 任意数值，常见值：1, 5, 10 |
| **参考高度线** | Y 轴参考电平 | 任意数值，单位 dBm |
| **灵敏度** | 接收灵敏度等级 | NORM/AUTO, NORM/HOLD, NORM, MID, HIGH1, HIGH2, HIGH3 |

输入框具有实时校验功能：
- 波长或分辨率输入不合法时，输入框边框变红并在下方显示错误提示
- 点击 **应用设置** 时会再次校验，无效参数会弹出警告对话框

> **注意：** 应用设置会先执行 `init` 命令（清空当前迹线并开始新的扫描），然后再逐项设置参数。

### 3. 光谱捕获

1. 在 **迹线** 输入框中填写要读取的迹线名称（默认 `TRA`，可选 `TRB`、`TRC` 等）
2. 点击 **捕获**
3. 程序会从设备读取该迹线的波长 (X) 和功率 (Y) 数据
4. 数据自动绘制在右侧 Matplotlib 图表中
5. 捕获记录自动添加到左侧历史列表

Matplotlib 工具栏功能（图表下方）：
- 缩放、平移、自适应
- 保存图表为 PNG/PDF/SVG 等格式

### 4. 设备截图

1. 点击 **截图**
2. 首次使用会弹出目录选择对话框，选择 BMP 文件保存位置
3. 程序会命令设备将当前屏幕截图保存到设备内存，再读取回本地
4. 截图自动显示在右侧预览区，同时添加到左侧历史列表
5. 后续截图会自动保存到同一目录，无需重复选择

截图过程约需 5-6 秒（含设备端保存和网络传输）。

> **提示：** 截图目录在同一会话内只选择一次。如需更换目录，需重启程序。

### 扫描控制

"捕获与截图"页按钮行最左侧新增三个扫描控制按钮，专为连续扫描场景设计 —— 当你在不停扫描的过程中
发现一个理想光谱值时，可以立刻让设备停下，再从容地捕获 / 截图。三个按钮以颜色区分语义：

| 按钮 | 颜色 | 功能说明 | 底层 SCPI |
|------|------|---------|-----------|
| **开始扫描** | 🟢 绿 | 按当前扫描模式（单次 / 连续 / 自动）启动或恢复扫描。停止后想继续时点它，无需重新下发参数。 | `:INIT` |
| **暂停** | 🟡 琥珀 | **等当前这一次扫描走完后再停**。停下时曲线是完整的，适合"抓住好值"。点击后状态栏显示"暂停中…"，按钮变灰，扫描完成即自动停止。 | 轮询 `:STATus:OPERation:EVENt?`（bit0=扫描完成），完成后发 `:ABORt` |
| **立即停止** | 🔴 红 | **立刻中止扫描**，不等当前扫描完成。也可在"暂停"等待过程中点它，直接打断等待并马上停止。 | `:ABORt` |

**典型工作流：**

1. 应用设置并选择"连续"模式，设备开始反复扫描，曲线实时刷新。
2. 看到满意的光谱 → 点 **暂停**，等当前这次扫完后停在完整曲线上。
3. 点 **捕获** / **截图** 保存该结果。
4. 继续观测 → 点 **开始扫描** 恢复连续扫描。

> **说明：**
> - "暂停"按钮在等待期间为禁用（灰）状态，停止后自动恢复。
> - 设备空闲（未在扫描）时点"暂停"，最多等待 60 秒后兜底停止；此时也可直接点"立即停止"退出等待。
> - 未连接设备时点任意按钮会提示"请先连接设备"。

### 5. 数据保存

先在左侧列表中 **点击选中** 要保存的记录，再点击 **保存**：

| 记录类型 | 保存格式 | 说明 |
|---------|---------|------|
| 光谱数据 | CSV | 两列：波长(m), 功率(dBm) |
| 设备截图 | BMP | 从临时文件复制到指定路径 |

### 6. 历史记录管理

- **单击** 左侧列表中的任意条目，右侧立即切换显示对应内容
- 光谱数据显示为折线图，截图数据显示为图片预览
- 点击 **清空** 可清除所有历史记录
- 列表中条目命名规则：
  - 光谱捕获：`捕获_YYYYMMDD_HHMMSS_迹线名`
  - 设备截图：`截图_YYYYMMDD_HHMMSS`

## 程序化调用 (osa 类)

如果不需要 GUI，可以直接在 Python 脚本中调用 `osa` 类：

### 基本用法

```python
from osa import osa

# 连接设备
client = osa("192.168.100.176", 10001)

# 查询设备 ID
print(client.idn)

# 设置参数
client.init(mode=1)              # 单次扫描模式
client.wl_center = 1550          # 中心波长 1550 nm
client.wl_span = 20              # 扫描范围 20 nm
client.resolution = 0.02         # 分辨率 0.02 nm
client.pdiv = 10                 # Y轴每格 10 dB
client.ylevel = -20              # 参考电平 -20 dBm
client.sensitivity = "HIGH1"     # 灵敏度

# 读取光谱数据
x, y = client.get_spectrum(trace="TRA")
# x: 波长数组 (m), y: 功率数组 (dBm)

# 截图
client.save_screenshot()                              # 保存到设备内存
bmp_data = client.read_device_file("screenshot")      # 读取 BMP 二进制数据
with open("output.bmp", "wb") as f:
    f.write(bmp_data)

# 断开
client.close()
```

### 可用属性

| 属性 | 读写 | SCPI 命令 | 说明 |
|------|------|-----------|------|
| `wl_center` | 读/写 | `:SENSe:WAVelength:CENTer` | 中心波长 (nm) |
| `wl_span` | 读/写 | `:SENSe:WAVelength:SPAN` | 扫描范围 (nm) |
| `resolution` | 读/写 | `:SENS:BWID` | 分辨率 (nm) |
| `pdiv` | 读/写 | `:DISP:TRAC:Y1:PDIV` | Y 轴每格刻度 (dB) |
| `ylevel` | 读/写 | `:DISP:TRAC:Y1:RLEV` | 参考电平 (dBm) |
| `sensitivity` | 读/写 | `:SENSe:SENSe` | 灵敏度等级 |
| `idn` | 只读 | `*IDN?` | 设备标识 |

### 可用方法

| 方法 | 说明 |
|------|------|
| `init(mode, trace)` | 初始化扫描。mode: 1=单次, 2=连续, 3=自动 |
| `start_sweep()` | 按当前扫描模式启动 / 恢复扫描（`:INIT`） |
| `abort()` | 立即中止扫描（`:ABORt`） |
| `poll_sweep_finished()` | 读取并清除"扫描完成"事件，返回是否完成过一次扫描 |
| `get_spectrum(trace)` | 读取指定迹线的光谱数据，返回 (波长数组, 功率数组) |
| `save_screenshot(filename)` | 将当前屏幕截图保存到设备内存 |
| `read_device_file(filename)` | 从设备内存读取文件，返回二进制数据 |
| `display(trace)` | 显示指定迹线 |
| `hide(trace)` | 隐藏指定迹线 |
| `close()` | 关闭连接 |
| `send(message)` | 发送原始 SCPI 命令（无返回值） |
| `query(message)` | 发送 SCPI 命令并接收返回值 |

## 通信协议

程序通过 TCP Socket 与光谱仪通信，使用 SCPI (Standard Commands for Programmable Instruments) 协议。

### 连接流程

```
客户端                              光谱仪
  |-- TCP Connect (IP:Port) -------->|
  |                                  |
  |-- "open "anonymous""  ---------->|  登录请求
  |<-- "AUTHENTICATE CRAM-MD5" ------|  认证方式
  |                                  |
  |-- "123456" ---------------------->|  密码
  |<-- "ready" ----------------------|  认证成功
  |                                  |
  |-- "*IDN?" ----------------------->|  查询设备ID
  |<-- "YOKOGAWA,AQ6370C,..." -------|  设备信息
```

### SCPI 命令示例

```
:FORMAT:DATA REAL,64          # 设置数据格式为 64 位浮点
:TRACe:ACTive TRA             # 激活迹线 A
:TRACE:X? TRA                 # 查询迹线 A 的波长数据
:TRACE:Y? TRA                 # 查询迹线 A 的功率数据
:MMEMory:STORe:GRAPhics ...   # 截图命令
:MMEMory:DATA? "file.BMP"     # 读取设备文件
```

### 数据格式

- 文本响应以 `\r\n` 结尾
- 二进制数据使用 IEEE 488.2 定义块格式：`#<n><count><data>\r\n`
  - `n`：后面 count 的位数（如 `5` 表示 count 是 5 位数）
  - `count`：数据的字节数
  - `data`：原始二进制数据

## 常见问题

**Q: 连接超时怎么办？**

A: 按以下顺序排查：
1. 确认光谱仪已开机且启动完成
2. `ping` 设备 IP 确认网络连通
3. 确认 IP 和端口号正确（在光谱仪的通信设置中可查看）
4. 检查防火墙是否阻止了连接

**Q: 截图失败怎么办？**

A: 截图过程需要约 5 秒，期间请勿操作设备。如果持续失败，检查日志窗口中的错误信息。常见原因是设备内存已满，需要在设备端清理不需要的文件。

**Q: 应用设置后光谱没有更新？**

A: 应用设置会初始化扫描，但数据采集需要时间（取决于扫描范围和分辨率）。等待设备扫描完成后，再点击 **捕获** 读取数据。

**Q: 连续扫描时怎么停下来抓取某一帧数据？**

A: 用按钮行的扫描控制按钮：看到理想光谱时点 **暂停**，设备会在当前这次扫描走完后停在完整曲线上，再点 **捕获** / **截图** 保存；想继续观测时点 **开始扫描**。若不想等当前扫描走完，点 **立即停止** 可立刻中止。详见 [扫描控制](#扫描控制)。

**Q: 保存的 CSV 文件中波长单位是什么？**

A: CSV 中的波长单位是**米 (m)**。例如 1550 nm 在 CSV 中显示为 `1.55e-06`。如需转换为 nm，乘以 `1e9` 即可。

**Q: 运行时提示 ModuleNotFoundError？**

A: 确认已安装所有依赖：
```bash
pip install -r requirements.txt
```

**Q: 连接时勾选"连接时重置"会有什么影响？**

A: 会执行 `*RST` 复位命令，清除设备上所有的扫描数据和用户设置，恢复出厂默认值。通常在需要完全重新配置时才勾选。
